<h1>ML for Systems </h1>

<p>Our vision for research on ML for Systems is laid out in <a href="http://cidrdb.org/cidr2019/papers/p117-kraska-cidr19.pdf">SageDB</a>, a new type of data processing system that highly specializes to a particular application through code synthesis and machine learning.
This vision is also a focus of <a href="http://dsail.csail.mit.edu/">MIT DSAIL</a>.</p>
<p>Here, we provide an overview of data systems components that we are currently working on, with more detailed project descriptions in the links, as well as a list of open-source repositories.</p>
<p>For high-level descriptions of our research, you can check out our <a href="https://learnedsystems.mit.edu/">Learned Systems Blog</a>.</p>
<p>If you want to find out more about the exciting work in this area, we have also compiled a list of <Link to="/projects/mlforsystems/papers/">ML for Systems Papers</Link>.</p>

<h2 class='bd-title'>Data Access </h2>

<StyledBlockQuote><p>Machine Learning just ate Algorithms in one large bite, thx to @tim_kraska, @alexbeutel, @edchi, @JeffDean & Polyzotis at @Googleâ€”faster, smaller trees, hashes, bloom filters</p>&mdash; Christopher Manning, Professor at Stanford</StyledBlockQuote>
  
<p>Storage layout and index structures are the most important factors to guarantee efficient data access, and both
are amenable to be enhanced by learned data and workload models. The original work on <a href="https://arxiv.org/pdf/1712.01208.pdf">learned indexes</a> showed that replacing traditional index structures with learned models can improve data access times while reducing memory footprint. Since then, we have worked to make learned indexes <a href="https://dl.acm.org/doi/pdf/10.1145/3318464.3389711">more practical</a>, developed <a href="https://arxiv.org/pdf/2006.13282.pdf">learned multi-dimensional indexes</a>, created a <a href="https://arxiv.org/pdf/2006.12804.pdf">benchmark for learned index structures</a>, and applied learned indexes to <a href="https://arxiv.org/pdf/1910.04728.pdf">DNA sequence search</a>.</p>

<h2 class='bd-title'>Query Execution </h2>
<p>Learning the data distribution can also be used to speed up query execution. In particular, we are applying learned techniques to <a href="https://dl.acm.org/doi/pdf/10.1145/3318464.3389752">sorting</a>, <a href="https://web.mit.edu/decima/content/sigcomm-2019.pdf">scheduling</a>, and joins.</p>
<p>Publications:</p>
<ul>
  <li><a href="https://web.mit.edu/decima/content/sigcomm-2019.pdf">Learning Scheduling Algorithms for Data Processing Clusters. SIGCOMM 2019.</a> Hongzi Mao, Malte Schwarzkopf, Shaileshh Bojja Venkatakrishnan, Zili Meng, Mohammad Alizadeh</li>
  <li><a href="https://dl.acm.org/doi/pdf/10.1145/3318464.3389752">The Case for a Learned Sorting Algorithm. SIGMOD 2020.</a> Ani Kristo*, Kapil Vaidya*, Ugur Cetintemel, Sanchit Misra, Tim Kraska</li>
  <li><a href="https://dl.acm.org/doi/abs/10.1145/3514221.3526158">LSched: A Workload-Aware Learned Query Scheduler for Analytical Database Systems. SIGMOD 2022.</a> Ibrahim Sabek, Tenzin Samten Ukyab, Tim Kraska</li>
</ul>
<h2 class='bd-title'>Query Optimization </h2>
<p>Traditional query optimizers are extremely hard to build,
maintain, and often yield sub-optimal query plans. The brittleness and complexity of the optimizer makes it a good candidate to be learned. We have introduced the first <a href="http://www.vldb.org/pvldb/vol12/p1705-marcus.pdf">end-to-end learned</a> <a href="https://arxiv.org/pdf/2004.03814.pdf">query optimizer</a>. We are also exploring <a href="https://ieeexplore.ieee.org/stamp/stamp.jsp?tp=&arnumber=9094107">learning-based cardinality estimation</a> techniques.</p>
<p>Publications:</p>
<ul>
  <li><a href="http://www.vldb.org/pvldb/vol12/p1705-marcus.pdf">Neo: A Learned Query Optimizer. VLDB 2019.</a> Ryan Marcus, Parimarjan Negi, Hongzi Mao, Chi Zhang, Mohammad Alizadeh, Tim Kraska, Olga Papaemmanouil, Nesime Tatbul</li>
  <li><a href="https://ieeexplore.ieee.org/stamp/stamp.jsp?tp=&arnumber=9094107">Cost-Guided Cardinality Estimation: Focus Where it Matters. SMDB 2020.</a> Parimarjan Negi, Ryan Marcus, Hongzi Mao, Nesime Tatbul, Tim Kraska, Mohammad Alizadeh</li>
  <li><a href="https://dl.acm.org/doi/pdf/10.1145/3448016.3452838">Bao: Making Learned Query Optimization Practical. SIGMOD 2021.</a> Ryan Marcus, Parimarjan Negi, Hongzi Mao, Nesime Tatbul, Mohammad Alizadeh, Tim Kraska</li>
